# AGENTS 지침

## 프로젝트 맥락
- 이 프로젝트는 스프링 입문 미션을 단계별로 수행하며 학습하는 것을 목표로 합니다.
- 모든 설명과 산출물은 한국어로 작성합니다.
- 코드는 미션 단위(`mission-xx-name`) → 테스크 단위(`task-xx-name`) 패키지로 나눕니다(예: `mission01_spring_intro.task01_hello`). 문서도 동일한 미션/테스크 단위로 정리합니다.

## 작업 진행 방식
- 사용자가 요청하는 하위 테스크를 하나씩 순서대로 처리합니다. 범위가 모호하면 간단히 확인하고 진행합니다.
- 작업 중에는 스프링을 처음 접하는 개발자의 시각에서 이해하기 쉽게 표현하고, 필요할 때만 짧은 주석을 추가합니다.

## 문서화 지침
- 각 테스크 완료 후 `docs/mission-xx-<name>/task-xx-<name>/README.md`에 보고서 형태로 기록합니다. 형식과 톤은 `docs/mission-01-spring-intro/task-01-hello/README.md`를 참고합니다.
- 문서에는 다음을 포함합니다: 작업 개요, 구현 단계와 주요 코드 경로/블록, 실행·빌드·테스트 방법과 예상 결과(명령 예시 포함), 결과 확인 방법(필요 시 스크린샷 안내).
- 학습 내용 섹션을 별도로 두고, 스프링 초심자가 이해하기 쉬운 용어로 핵심 개념(어노테이션, 의존성, 흐름 등)을 설명합니다. 왜 그렇게 동작하는지와 사용 이유를 함께 적습니다.
- 보고서 어조로 구현 과정과 결과를 정리하고, 동일한 섹션 구조와 서식을 유지합니다.

## 추가 원칙
- 사용자 요청 범위를 넘는 변경은 하지 않습니다.
- 새 파일이나 문서를 만들 때는 위치와 역할을 명확히 드러내고, 기존 문서 스타일을 유지합니다.
- 미션/테스크 작업이 끝나면 루트 `README.md`에 미션·테스크 인덱스 링크를 추가/갱신해 탐색 경로를 명확히 합니다.
