# 네트워크 계층 이해하기

`mission-03-http-web-basic`의 `task-10-network-layers` 보고서입니다. OSI 7계층 모델에서 각 계층이 어떤 역할을 하는지 정리하고, 데이터가 계층을 따라 어떻게 이동하는지 설명했습니다. 이어서 TCP/IP 4계층 모델과 비교해 실제 인터넷 통신에서 어떻게 대응되는지도 함께 정리했습니다.

## 1. 작업 개요

- 미션/태스크: `mission-03-http-web-basic` / `task-10-network-layers`
- 목표:
  - OSI 7계층 모델의 각 계층 역할과 기능을 이해한다.
  - 계층 간 데이터 흐름과 캡슐화 관점을 정리한다.
  - TCP/IP 4계층 모델과 OSI 7계층 모델의 대응 관계를 비교한다.

## 2. 주요 해설

### 2.1 OSI 7계층 설명표

| 계층 | 이름 | 역할 | 대표 예시 |
|---|---|---|---|
| 7계층 | 응용 계층(Application) | 사용자와 가장 가까운 계층으로, 애플리케이션이 네트워크 서비스를 사용하게 합니다. | HTTP, SMTP, FTP, DNS |
| 6계층 | 표현 계층(Presentation) | 데이터 형식 변환, 인코딩, 암호화, 압축을 담당합니다. | TLS의 암호화 개념, 문자 인코딩 |
| 5계층 | 세션 계층(Session) | 통신 세션의 생성, 유지, 종료를 다룹니다. | 세션 연결 관리 개념 |
| 4계층 | 전송 계층(Transport) | 종단 간 데이터 전달, 신뢰성, 흐름 제어, 포트 기반 구분을 담당합니다. | TCP, UDP |
| 3계층 | 네트워크 계층(Network) | 목적지까지의 경로 선택과 논리 주소(IP)를 기반으로 패킷 전달을 담당합니다. | IP, 라우터 |
| 2계층 | 데이터링크 계층(Data Link) | 같은 네트워크 안에서 프레임 단위 전송과 MAC 주소 기반 전달을 담당합니다. | Ethernet, 스위치 |
| 1계층 | 물리 계층(Physical) | 전기 신호, 광 신호, 무선 신호처럼 실제 비트를 매체에 실어 보냅니다. | 케이블, 허브, 전파 |

### 2.2 계층 간 데이터 흐름

- 데이터를 보내는 쪽에서는 상위 계층에서 하위 계층으로 내려가며 각 계층 정보가 덧붙습니다.
- 데이터를 받는 쪽에서는 반대로 하위 계층에서 상위 계층으로 올라가며 각 계층 정보가 제거되고 원본 데이터가 복원됩니다.
- 이 과정을 보통 `캡슐화(Encapsulation)`와 `역캡슐화(Decapsulation)`라고 부릅니다.

#### 예시 흐름: 브라우저에서 HTTP 요청 전송

1. 응용 계층에서 `GET /notes` 같은 HTTP 메시지를 만듭니다.
2. 전송 계층에서 TCP 헤더가 붙어 세그먼트가 됩니다.
3. 네트워크 계층에서 IP 헤더가 붙어 패킷이 됩니다.
4. 데이터링크 계층에서 MAC 주소 정보가 붙어 프레임이 됩니다.
5. 물리 계층에서 비트 신호로 변환되어 케이블이나 무선으로 전송됩니다.
6. 서버는 이 과정을 반대로 따라 올라가며 최종적으로 HTTP 요청을 해석합니다.

### 2.3 OSI 7계층과 TCP/IP 4계층 비교표

| OSI 7계층 | TCP/IP 4계층 | 비교 설명 |
|---|---|---|
| 응용 계층 | 응용 계층 | TCP/IP에서는 응용, 표현, 세션을 하나의 응용 계층으로 묶어 다루는 경우가 많습니다. |
| 표현 계층 | 응용 계층 | 데이터 표현, 인코딩, 암호화 같은 역할이 응용 계층 안에서 함께 처리됩니다. |
| 세션 계층 | 응용 계층 | 세션 유지 책임도 애플리케이션 프로토콜이나 라이브러리 수준에서 처리됩니다. |
| 전송 계층 | 전송 계층 | 두 모델 모두 종단 간 전달과 포트 기반 구분을 담당합니다. |
| 네트워크 계층 | 인터넷 계층 | IP 기반 라우팅과 목적지 전달 역할이 대응됩니다. |
| 데이터링크 계층 | 네트워크 인터페이스 계층 | 로컬 네트워크 전송과 프레임 처리가 포함됩니다. |
| 물리 계층 | 네트워크 인터페이스 계층 | 실제 신호 전송까지 함께 포함해 다루는 경우가 많습니다. |

### 2.4 TCP/IP 4계층 설명표

| 계층 | 역할 | 대표 예시 | OSI와의 관계 |
|---|---|---|---|
| 응용 계층 | 애플리케이션 프로토콜 처리 | HTTP, DNS, SMTP | OSI 5~7계층을 묶은 개념에 가깝습니다. |
| 전송 계층 | 종단 간 전달, 포트, 신뢰성 제어 | TCP, UDP | OSI 4계층과 거의 대응됩니다. |
| 인터넷 계층 | IP 주소 기반 전달과 라우팅 | IP, ICMP | OSI 3계층과 대응됩니다. |
| 네트워크 인터페이스 계층 | 실제 링크 전송과 물리 매체 처리 | Ethernet, Wi-Fi | OSI 1~2계층을 묶은 개념에 가깝습니다. |

### 2.5 왜 두 모델을 함께 보는가

1. OSI 7계층은 역할을 세밀하게 나눠 학습하기 좋습니다.
2. TCP/IP 4계층은 실제 인터넷 프로토콜 구조와 더 가깝습니다.
3. 그래서 개념을 배울 때는 OSI 7계층이, 실전 네트워크 이해에는 TCP/IP 모델이 자주 사용됩니다.

## 3. 새로 나온 개념 정리 + 참고 링크

- **OSI 7계층**
  - 핵심: 네트워크 통신을 7개의 역할 계층으로 나눠 설명하는 개념 모델입니다.
  - 왜 쓰는가: 복잡한 네트워크 동작을 단계별 책임으로 나눠 이해하고 문제를 분석하기 쉽기 때문입니다.
  - 참고 링크:
    - IBM OSI model overview: https://www.ibm.com/think/topics/osi-model
    - Cloudflare OSI model: https://www.cloudflare.com/learning/ddos/glossary/open-systems-interconnection-model-osi/

- **TCP/IP 4계층**
  - 핵심: 실제 인터넷 프로토콜 구조를 설명할 때 널리 쓰이는 계층 모델입니다.
  - 왜 쓰는가: HTTP, TCP, IP, Ethernet 같은 실제 프로토콜이 어느 층에서 동작하는지 더 직접적으로 연결해 볼 수 있기 때문입니다.
  - 참고 링크:
    - RFC 1122 Requirements for Internet Hosts: https://datatracker.ietf.org/doc/html/rfc1122
    - Cloudflare TCP/IP model: https://www.cloudflare.com/learning/ddos/glossary/tcp-ip/

- **캡슐화와 역캡슐화**
  - 핵심: 데이터를 보낼 때는 각 계층 헤더를 덧붙이고, 받을 때는 반대로 제거하는 과정입니다.
  - 왜 쓰는가: 각 계층이 자신의 역할에 필요한 정보만 추가하고, 나머지는 다음 계층에 위임하는 구조를 이해할 수 있기 때문입니다.
  - 참고 링크:
    - IBM OSI model overview: https://www.ibm.com/think/topics/osi-model

- **전송 계층과 네트워크 계층의 차이**
  - 핵심: 전송 계층은 종단 간 전달을, 네트워크 계층은 목적지까지의 경로 전달을 담당합니다.
  - 왜 쓰는가: TCP와 IP의 역할을 구분해야 HTTP 요청이 실제로 어떻게 전달되는지 이해할 수 있기 때문입니다.
  - 참고 링크:
    - RFC 793 TCP: https://datatracker.ietf.org/doc/html/rfc793
    - RFC 791 IP: https://datatracker.ietf.org/doc/html/rfc791

## 4. 학습 내용

- OSI 7계층은 네트워크를 공부할 때 "누가 무엇을 담당하는가"를 구조적으로 보여 주는 좋은 틀입니다.
- 실제 인터넷에서는 TCP/IP 4계층 모델이 더 자주 쓰이지만, 그 안의 역할을 잘게 이해하려면 OSI 모델이 더 설명적입니다.
- 브라우저에서 HTTP 요청 하나를 보내는 단순한 동작도, 실제로는 응용 계층부터 물리 계층까지 여러 단계를 거쳐 전송됩니다.
- 계층을 나눠 이해하면 문제가 생겼을 때 애플리케이션 문제인지, TCP 문제인지, IP 라우팅 문제인지 더 체계적으로 추적할 수 있습니다.
