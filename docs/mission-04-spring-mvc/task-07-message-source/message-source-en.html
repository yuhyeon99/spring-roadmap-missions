Internationalization with MessageSource

Switch between Korean and English using the lang parameter and message bundles.

Spring MessageSource selects the message file for the current locale, so the same view can render different text depending on the language.

Current locale: en name=Spring Learner
Hello, Spring Learner. This page is currently using the English message bundle.
View Korean View English

Why the screen changes

When LocaleChangeInterceptor reads the lang parameter and stores it in LocaleResolver, MessageSource chooses messages_ko.properties or messages_en.properties.

GET /mission04/task07/messages?lang=ko&name=MVC%20학습자
GET /mission04/task07/messages?lang=en&name=Spring%20Learner

Practice point

Call the same URL with ?lang=ko and ?lang=en to see the same template render different messages.
